# AVATAR OnE 기능명세서 변경사항 (v1.9 → v1.10)

> **작성일**: 2026-04-07
> **범위**: Trainer 워크로드-리소스 통합, 데드 코드 정리
> **대상**: 외주 개발업체 전달용

---

## 1. Trainer 탭 구조 변경

### 기존 (v1.9)

```
관리자 모드 Trainer: 요청 관리 | 워크로드 | 리소스 | Solver 관리 | 모델 선택
```

### 변경 (v1.10)

```
관리자 모드 Trainer: 요청 관리 | 워크로드 | Solver 관리 | 모델 선택
```

| 변경 | 내용 |
|------|------|
| "리소스" 탭 제거 | 독립 탭으로 존재하던 클러스터 리소스 현황 화면을 제거 |
| "워크로드" 탭에 통합 | 워크로드 탭 상단에 클러스터 전체 자원 현황을 표시 |

**변경 이유**: 관리자가 워크로드 실행 상태를 모니터링하면서 동시에 클러스터 자원 현황을 파악할 수 있도록 한다. 기존에는 탭을 전환해야 했으나, 통합 후에는 하나의 화면에서 모든 정보를 확인할 수 있다.

---

## 2. 워크로드 화면 자원 현황 통합

### 변경 전

워크로드 탭:
```
┌─────────────────────────────────────────┐
│ 워크로드 목록 테이블                      │
│  워크로드명 | 상태 | 우선순위 | 자원 | ... │
├─────────────────────────────────────────┤
│ (클릭 시) 학습 진행 차트 + 컴포넌트 실행  │
└─────────────────────────────────────────┘
```

### 변경 후

워크로드 탭:
```
┌─────────────────────────────────────────┐
│ 클러스터 자원 현황 (NEW)                  │
│  GPU A100 [████████░░░░] 12/16기 유휴 4  │
│  GPU V100 [██████░░░░░░] 6/8기 유휴 2    │
│  메모리   [█████░░░░░░░] 1200/2048GB     │
│  CPU     [███░░░░░░░░░] 80/256코어       │
├─────────────────────────────────────────┤
│ 워크로드 목록 테이블                      │
│  워크로드명 | 상태 | 우선순위 | 자원 | ... │
├─────────────────────────────────────────┤
│ (클릭 시) 학습 진행 차트 + 컴포넌트 실행  │
└─────────────────────────────────────────┘
```

### 세부 사항

| 항목 | 내용 |
|------|------|
| 자원 현황 위치 | 워크로드 테이블 상단 (2x2 그리드) |
| 표시 자원 | GPU A100, GPU V100, 메모리, CPU |
| 표시 정보 | 총량, 요청량, 사용량, 유휴량 (바 차트) |
| 데이터 소스 | 실행 중(running) + 대기(queued) 워크로드의 자원 합산 |
| 워크로드 테이블 | 기존과 동일 (변경 없음) |
| 컴포넌트별 실행 현황 | 기존과 동일 — 워크로드 클릭 시 표시 (변경 없음) |

## 3. 코드 정리 (Legacy 컴포넌트 삭제)

v1.9에서 메뉴 구조 변경으로 사용되지 않게 된 레거시 컴포넌트를 삭제한다.

### 삭제 대상

| 컴포넌트 | 이전 역할 | 삭제 이유 |
|---------|----------|----------|
| `HomePage` | 대시보드 (통계 카드 + 워크플로우 다이어그램) | v1.9에서 메뉴 제거됨. Builder/Operator로 대체 |
| `TestRunPage` | 독립 테스트 실행 화면 | Builder 학습 요청 탭의 사전 테스트로 통합됨 |
| `ComponentLibraryPage` | 독립 컴포넌트 라이브러리 화면 | Builder 내 인라인 라이브러리로 통합됨 |
| `TrainerPage` | 독립 학습 요청 화면 (사용자용) | Builder 학습 요청 탭으로 통합됨 |
| `QueuePage` | 대기열 관리 (관리자) | Trainer 요청 관리로 통합됨 |
| `WorkloadsPage` | 워크로드 목록 (기존 버전) | `WorkloadsPageWithResources`로 대체됨 |
| `ModelsPage` | 결과 모델 목록 (기존 버전) | Trainer 모델 선택 탭으로 대체됨 |
| `ResourcesPage` | 리소스 현황 (독립 탭) | 워크로드 탭에 통합됨 (이번 변경) |

### 보존 컴포넌트 (삭제하지 않음)

| 컴포넌트 | 이유 |
|---------|------|
| `ApprovalPage` | Trainer "요청 관리" 탭에서 활성 사용 중 |
| `WorkloadsPageWithResources` | Trainer "워크로드" 탭의 현재 구현체 |

# Change Summary: v1.9 → v1.10 (English)

> **Date**: 2026-04-07
> **Scope**: Trainer workload-resource consolidation, dead code cleanup

---

## 1. Trainer Tab Structure Change

### Before (v1.9)

```
Admin Trainer: Request Mgmt | Workloads | Resources | Solver Mgmt | Model Selection
```

### After (v1.10)

```
Admin Trainer: Request Mgmt | Workloads | Solver Mgmt | Model Selection
```

The standalone "Resources" tab is removed. Its content (cluster resource utilization bars) is now displayed at the top of the "Workloads" tab.

**Reason**: Administrators can now monitor workload execution and cluster resource utilization in a single view without switching tabs.

---

## 2. Workload View — Integrated Resource Overview

The workload tab now shows cluster-wide resource utilization (GPU A100, GPU V100, Memory, CPU) in a 2x2 bar chart grid above the workload table.

| Item | Detail |
|------|--------|
| Position | Top of workload tab, above the workload table |
| Resources shown | GPU A100, GPU V100, Memory, CPU |
| Data displayed | Total, requested, used, idle (bar chart) |
| Data source | Sum of running + queued workloads' resource allocations |
| Workload table | Unchanged |
| Per-component execution status | Unchanged — shown on workload click |

---

## 3. Legacy Component Cleanup

Eight unused legacy components are removed from `App.jsx`:

| Component | Previous Role | Reason for Removal |
|-----------|--------------|-------------------|
| `HomePage` | Dashboard | Replaced by Builder/Operator in v1.9 |
| `TestRunPage` | Standalone test execution | Integrated into Builder pre-test |
| `ComponentLibraryPage` | Standalone component library | Integrated into Builder inline library |
| `TrainerPage` | User training request | Integrated into Builder training request tab |
| `QueuePage` | Queue management (admin) | Integrated into Trainer request management |
| `WorkloadsPage` | Workload list (old version) | Replaced by `WorkloadsPageWithResources` |
| `ModelsPage` | Model list (old version) | Replaced by Trainer model selection tab |
| `ResourcesPage` | Resource overview (standalone tab) | Integrated into Workloads tab (this change) |

### Preserved Components (NOT deleted)

| Component | Reason |
|-----------|--------|
| `ApprovalPage` | Actively used in Trainer "Request Mgmt" tab |
| `WorkloadsPageWithResources` | Current workload tab implementation |
